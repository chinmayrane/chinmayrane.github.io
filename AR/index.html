What is it?


“Augmented reality (AR) is a term for a live direct or indirect view of a physical, real-world environment whose elements are augmented by virtual computer-generated sensory input, such as sound or graphics. It is related to a more general concept called mediated reality, in which a view of reality is modified (possibly even diminished rather than augmented) by a computer. As a result, the technology functions by enhancing one’s current perception of reality.”

What we can do?

  •   Create a new AR service.
  •   Use existing AR Service such as Layar.
  •   Layar  supports only Android and iOS, no other platform.
  •   Cross platform development is a steep curve.
  •   Make a custom AR Service.


 

   Lot of Physics engine, Game engines can be used to add a gamification concept along with the traditional AR activities.



Components

  • Camera Data

  • Location Data

  • Sensor Data


Most solutions use OpenCV Library, which has existing solutions of the most common AR problems.


Usage


  •   Ultimate goal is to draw over the Camera Feed in real Time.
  •   Can be done as a Bitmap or a texture on a 3D Surface
  •  Android has android.graphics and android.opengl which helps us achieve this redraw in each frame.
  •  The platform specificity comes as a barrier to development

    Google Glass Mirror API is a complete package but only limited

   to Google Glass like devices.

Augmentation Data

  •  So where does the Augmentation Data come from?
    • Our own custom database (e.g Zomato's DB of restaurants)
    • This can be accessed over the PaaS(cloud) platform.
    •  Else, can use the existing WebservicesAPI provided by the proprietary vendors and consume them.


             Generally, the trend suggest that the application has a

           preloaded augmentation data DB which ideally is SQLite
