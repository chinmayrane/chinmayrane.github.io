Two way Data Binding


   -   Two way binding between the data and the model

   -   Model updates  the view and view updates the model

   -   Binded together through $scope.

   -   Saves a lot of rework   unlike other frameworks -jQuery

   -   Code is lot simpler to follow.

   -    No heavy DOM Manipulation

   -    No event based functions.




MVC of Angular


  - Model : A object must be referenced by Angular Scope as a  

                       property of that scope object.

  - ViewModel : A viewmodel is an object that provides specific

                                    data and methods to maintain specific views.

  - View :  The views rendered on the browsers, Angular makes

                      them understand through ng-controllers and

                      ng-view.

  - Controller: Initiate state of the $scope object and to augment
                                and add behaviour to it.

                       

* Pro Tip - Don't use controller for DOM Manipulation

  -  Created using the new operator
  -  Can be looked up by referring to a global variable.
  -  Can be passed into where it is needed.
  -  Used in controllers and factory methods.

Standard Services

 -  $http

     Sending AJAX requests and getting data in JSON form 

$http({ method: 'GET', url: '/foo' }).
  success(function (data, status, headers, config) {
    // ...
  }).
  error(function (data, status, headers, config) {
    // ...
  });

 - $resource

      Access to REST services in a more high level fashion 

var Users = $resource('/user/:userId', { userId: '@id' });
var user = Users.get({ userId: 123 }, function () {
  user.abc = true;
  user.$save();
});
 - RESTangular : Third party  solution for declarative and abstracts more of details away .

Routing

  - Declared by $routeProvider, which provides $route service.

  - Binds the Controllers, views, and the URLs 

  - Can use deep linking which uses the browsers history

  - This doesn't solve the problem of script load ordering or lazy loading.

  - AMD/ requireJS can be used independently and can exist side by side. 

var myApp =  angular.module('myAppModule', ['ngRoute',myAppControllers']);

myApp.config(['$routeProvider',function($routeProvider){
   $routeProvider.when('/home',{
         templateURL : 'partials/homeView.html',
         controller : 'ControllerHome'
}).when('/aboutus', {
         templateURL : 'partials/aboutus.html',
         controller : 'ControllerAbout'
}).otherwise(
           redirectTo: '/home'});
}]);

Code Structure


- js

- partials  //partials folder holdinf views
  == view1.html
  == view2.html

- index.html //main file which has the ng-app declarations.

- css
 == main.css //affecting any of the files.
 == bootstrap.css //if any bootstrapping library used.
